# Case studies

Optional evidence-backed notes that show how principles appear in a named system.

This upstream repository does **not** host filled case studies. Copy this category into your private instance and publish named systems only when you intend them to be public.

## When to use

- The lesson requires a **named system** and repo-relative evidence.
- A principle alone is not enough to convey how the pattern was applied.

## Hard rules

- No secrets, credentials, or copied source blobs
- Prefer principles when the lesson does not need a named org
- Each org or product gets its own subdirectory only when intentionally published
